What is CI/CD?

Continuous Integration and Continuous Delivery (CI/CD) denotes a set of disciplined engineering practices that enable software teams to deliver value continuously and reliably. CI emphasizes the frequent integration of code changes into a shared repository, where each change is automatically built and verified by automated tests. CD extends this process by automating the deployment of validated artifacts to staging or production environments, reducing manual intervention and minimizing release risk.

What happens after a Push? (Pipeline)

Once changes are pushed to the remote repository, the CI/CD pipeline is triggered and executes a defined sequence of stages. Typical stages include checking out the source code, building the application, running automated tests, and deploying the resulting artifacts to the designated environment. In GitHub Actions, these stages are defined in a workflow file that orchestrates the job execution automatically.
